# ADR-0001: SKILL.md 指令正文保持中文，以内联双语语言协议实现输出自适应

## 背景

spec-dev 需要中英双语适配：英文用户应获得全英文交互，但插件约 4600 行 skill 指令、参考文档为中文，且经 7.x 多轮措辞校准，语义精密。

## 决定

指令正文不翻译，保持中文单一源；在 8 个 SKILL.md 顶部内联统一的英前中后双语「语言协议块」，声明输出语言判定优先级（显式指定 > 输入语言 > 默认英语）、产物语言规则与固定话术的语义模板原则；配套清除确定性中文输出面（话术、descriptions、模板、README、snippet、脚本文案）。

## 理由

翻译 4600 行会扰动已校准的指令语义、diff 不可审，且与项目 5.4.0 确立的「单一源」哲学冲突（曾为消除双份漂移退役 check-mirrors 同步机制）；概率性的语言拖拽可由平台 `language` 设置与内联协议双信号压制，且本方案为全面英文化的真子集，evals 观测到英文输出夹杂频发时可增量升级、不返工。

**被否方案**：指令全面英文化（翻译漂移风险高、维护负担翻倍）；双份 SKILL.md 按 locale 分发（平台无此机制，双份漂移税与单一源哲学正面冲突）。
