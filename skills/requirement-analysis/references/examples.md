# 使用示例

## 复杂需求示例

**用户**: "我需要一个用户活动跟踪功能来记录用户操作，支持多租户隔离和实时分析"

**执行策略**：
- **阶段 1**: 使用 ultrathink 分析需求（多模块、复杂逻辑）
- **阶段 2**: 并行探索（3 个 agent：数据层、服务层、API层）
- **阶段 3**: 外部资源研究（context7 查询实时分析库）
- **阶段 4**: 澄清数据保留策略和操作类型
- **阶段 5**: 使用 ultrathink 设计架构
- **阶段 6**: 展示完整实施计划
- **阶段 7**: 实施开发
- **阶段 8**: 并行深度审查（5 个审查维度）
- **阶段 9**: 总结

- **阶段 9**: 总结

## 新技术栈示例

**用户**: "使用 Socket.io 实现实时通知功能"

**执行策略**：
- **阶段 1**: 识别需要使用第三方库
- **阶段 2**: 查找现有 WebSocket 代码
- **阶段 3**: 外部资源研究（context7 查询 Socket.io 文档）
- **阶段 4**: 澄清通知类型和历史记录需求
- **阶段 5**: 基于 Socket.io 最新 API 设计方案
- **阶段 6**: 展示集成计划
- **阶段 7**: 实施开发
- **阶段 8**: 代码审查（确保正确使用 Socket.io API）
- **阶段 9**: 总结
